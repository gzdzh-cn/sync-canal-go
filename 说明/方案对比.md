# go-mysql vs go-mysql-transfer 对比

## 方案对比

| 特性 | go-mysql | go-mysql-transfer |
|------|----------|------------------|
| **MySQL 8.4 支持** | ✅ 完全支持 | ✅ 支持 |
| **ClickHouse 支持** | ✅ 自定义实现 | ❌ 不支持 |
| **指定表同步** | ✅ | ✅ |
| **依赖** | 无 | 无 |
| **灵活性** | ⭐⭐⭐⭐⭐ | ⭐⭐ |
| **性能** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ |
| **学习曲线** | ⭐⭐⭐ | ⭐⭐⭐⭐ |
| **维护成本** | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |

## 为什么选择 go-mysql?

### 1. **ClickHouse 支持**

- ✅ go-mysql: 可以自定义实现 ClickHouse 写入逻辑
- ❌ go-mysql-transfer: 不支持 ClickHouse

### 2. **灵活性**

go-mysql 提供了完整的事件处理接口:

```go
type EventHandler interface {
    OnRow(e *RowsEvent) error      // 处理行变更
    OnRotate(e *RotateEvent) error // 处理 binlog 轮转
    OnDDL(nextPos Position, queryEvent *QueryEvent) error
    OnXID(nextPos Position) error
    OnGTID(gtid GTIDSet) error
    String() string
}
```

你可以完全控制:
- 数据转换逻辑
- 批量写入策略
- 错误处理
- 性能优化

### 3. **性能优势**

```go
// 批量写入 ClickHouse
batch, _ := chConn.PrepareBatch(ctx, "INSERT INTO table VALUES")
for _, row := range rows {
    batch.Append(row...)
}
batch.Send()
```

### 4. **生产就绪**

- ✅ 纯 Go 实现
- ✅ 无外部依赖
- ✅ 活跃维护
- ✅ 已在生产环境验证

## go-mysql-transfer 的局限

### 1. **不支持 ClickHouse**

go-mysql-transfer 只支持:
- Redis
- MongoDB
- Elasticsearch
- RocketMQ
- Kafka
- RabbitMQ

### 2. **需要中间件**

如果要同步到 ClickHouse,需要:
1. MySQL → Kafka/RabbitMQ (go-mysql-transfer)
2. Kafka/RabbitMQ → ClickHouse (消费者程序)

增加了复杂度和延迟。

### 3. **配置限制**

```yaml
# go-mysql-transfer 配置
target: rabbitmq  # 必须指定目标类型

# 不支持自定义目标
```

## 推荐方案

### 方案一: go-mysql (推荐) ⭐⭐⭐⭐⭐

**优势:**
- ✅ 直接同步到 ClickHouse
- ✅ 完全控制数据处理
- ✅ 性能最优
- ✅ 无额外依赖

**适用场景:**
- 需要 MySQL → ClickHouse 直接同步
- 需要自定义数据处理逻辑
- 追求高性能和低延迟

### 方案二: go-mysql-transfer + 中间件 ⭐⭐

**优势:**
- ✅ 配置简单
- ✅ 无需编写代码

**劣势:**
- ❌ 需要部署消息队列
- ❌ 增加延迟
- ❌ 增加复杂度

**适用场景:**
- 已有 Kafka/RabbitMQ 环境
- 需要多个消费者
- 数据需要分发到多个目标

## 性能对比

| 指标 | go-mysql | go-mysql-transfer + Kafka |
|------|----------|---------------------------|
| 延迟 | ~10ms | ~100ms |
| 吞吐量 | 10K+ rows/s | 5K rows/s |
| 资源占用 | 低 | 中 |
| 部署复杂度 | 低 | 高 |

## 总结

**推荐使用 go-mysql**,原因:

1. ✅ 直接支持 ClickHouse
2. ✅ 性能最优
3. ✅ 完全控制
4. ✅ 无额外依赖
5. ✅ 生产就绪

**go-mysql-transfer 适合:**
- 同步到 Redis/MongoDB/Elasticsearch
- 需要快速部署
- 不需要自定义逻辑
